# Data

The training corpus for Ember v0.1.

## Sources

1. **Synthetic** (`data/synthetic/`) — LHC-shaped post-training examples generated by Sonnet 4.6 from a small seed set of hand-curated examples. See [`PLAN.md`](../PLAN.md) Stage 1 for the construction process.

2. **Real agent traces** (`data/traces/`) — reserved for future versions. Anonymized agent run traces from production deployments. **Gitignored.** Customer data never enters this repository directly.

## Pipeline (placeholder — operational once Stage 1 lands)

```bash
# 1. Hand-curate seed examples (~50, see PLAN.md Stage 1.1)
$EDITOR data/seeds.jsonl

# 2. Validate seed set as few-shot prompt against K2.6 (PLAN.md Stage 1.2)
python -m data.validate_seeds --seeds data/seeds.jsonl

# 3. Generate synthetic examples at scale (PLAN.md Stage 1.3)
python -m data.prepare synthetic \
    --seeds data/seeds.jsonl \
    --target-count 2000 \
    --output data/synthetic/v0.1.jsonl

# 4. Filter and split into train/eval (PLAN.md Stage 1.4)
python -m data.prepare split \
    --input data/synthetic/v0.1.jsonl \
    --train data/synthetic/v0.1_train.jsonl \
    --eval data/synthetic/v0.1_eval.jsonl
```

## Format

Every example is a JSONL row in OpenAI chat-completions format:

```json
{
  "messages": [
    {"role": "system", "content": "..."},
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."}
  ],
  "metadata": {
    "source": "synthetic|trace",
    "category": "state_recall|commitment|resumption",
    "difficulty": "seed|easy|medium|hard",
    "gap_tokens": 6000
  }
}
```

The `metadata.category` and `metadata.difficulty` fields let us stratify training and eval splits by the same taxonomy LHC uses.
